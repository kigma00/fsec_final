PCI DSS(Payment Card Industry Data Security Standard) 체크리스트

1. PCI DSS 개요
   - 목적: 신용카드 정보 보안 표준
   - 적용범위: 신용카드 정보를 처리, 저장, 전송하는 모든 조직
   - 버전: PCI DSS v4.0 (2022년 3월 발표)

2. 6개 핵심 목표
   목표 1: 보안 네트워크 및 시스템 구축 및 유지
   목표 2: 카드 소유자 데이터 보호
   목표 3: 취약점 관리 프로그램 유지
   목표 4: 강력한 접근 통제 구현
   목표 5: 네트워크 모니터링 및 테스트 정기 수행
   목표 6: 정보보안 정책 수립

3. 12개 핵심 요구사항
   요구사항 1: 방화벽 설치 및 구성
   요구사항 2: 공급업체 기본 설정 변경
   요구사항 3: 카드 소유자 데이터 보호
   요구사항 4: 개방형 공용 네트워크에서 카드 소유자 데이터 암호화
   요구사항 5: 악성 소프트웨어 방지
   요구사항 6: 보안 시스템 및 애플리케이션 개발 및 유지
   요구사항 7: 비즈니스 필요성에 따른 접근 제한
   요구사항 8: 고유 ID 할당 및 접근 관리
   요구사항 9: 물리적 접근 제한
   요구사항 10: 모든 네트워크 리소스 및 카드 소유자 데이터 접근 추적 및 모니터링
   요구사항 11: 보안 시스템 및 프로세스 정기 테스트
   요구사항 12: 정보보안 정책 수립

4. 주요 체크리스트
   □ 방화벽 정책 및 절차 문서화
   □ 보안 설정 표준 수립
   □ 카드 데이터 암호화 구현
   □ 바이러스 백신 소프트웨어 설치
   □ 보안 패치 관리 프로세스 수립
   □ 접근 권한 정기 검토
   □ 보안 사고 대응 계획 수립

5. 인증 및 검증
   - 자체 평가: SAQ(Self-Assessment Questionnaire)
   - 외부 감사: QSA(Qualified Security Assessor)
   - 스캔 검증: ASV(Approved Scanning Vendor)
   - 인증 주기: 연 1회
